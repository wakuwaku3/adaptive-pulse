# プランニングと実装開始の合意 (常に適用)

「まずはプランニングしたい」のようにプランニング自体が依頼のとき、実装開始には**内容の合意**が必要 (FB 2026-07-22)。

- プラン提示 (plan mode の承認操作を含む) を実装 Go とみなさない。承認操作は「プランの方向性は良い」の意味でしかない。
- プラン提示後は、要件から派生する論点 (スコープ境界・未指定の挙動・データモデルの選択肢) を自分から出し尽くし、ユーザが「実装して」と明示するまで待つ。
- これは協働モデルの「確認質問は出さず進める」の例外である。プランニング依頼は「議論したい」という意思表示なので、自動着手の対象にしない。
