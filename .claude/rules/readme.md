# README

README は **人間向けの入り口**となる stock ドキュメント。public repo なので、読者は本人だけでなく通りすがりの閲覧者も想定する。

- 書くこと: プロジェクトの概要/目的、免責 (医療機器ではない)、ステータス、インストール (サイドロード) 手順、詳細 docs へのリンク。
- **開発情報は書かない**: 開発環境構築 (devbox/direnv/gradle 等)・開発フロー・コーディング規約・協働モデルは AI 向けで `CLAUDE.md` / `.claude/rules/` が持つ。
- 簡潔に保ち、詳細は一次情報へ委ねる: 要件・設計は `docs/stock/`、検討経緯は `docs/notes/`。二重管理せずリンクか要約一文に留める。
- リポジトリを見れば分かる情報 (ディレクトリ構成等) は固定化しない。stock として常に最新に保ち、インストール手順等が変わったら同じターンで直す。
