# ドキュメント

**すべての `.md` は stock か flow のいずれかに分類し、許可された場所にのみ置く。** それ以外の場所への `.md` 配置は禁止 (Stop hook と CI の `scripts/check_docs.sh` が検知する)。

- **stock**: 最新状態を上書きで保つ確定情報。許可される置き場所:
  - `README.md` (任意階層) — 概要
  - `CLAUDE.md` (任意階層) — repo ルール
  - `.claude/` 配下 — AI 向けの開発ルール等
  - `docs/stock/` 配下 — 上記以外の確定ドキュメント (要件・設計など)
- **flow**: 時系列で追記し消さない検討経緯・調査ログ・運用ジャーナル。許可される置き場所:
  - `docs/notes/YYYYMMDD__<topic>/index.md`
- 検討は flow で行い、確定したら stock (`docs/stock/` または該当ルール) に昇格させ、経緯は flow に残す。
- **stock は経緯を知らなくても読める肯定形で書く**。廃案・不採用の代替案への否定形参照 (「〜は使わない」「〜は廃止」「〜不要」等、その案を検討した経緯を知らないと意味が取れない記述) を stock に書かない。「なぜそうしたか／なぜ別案を採らなかったか」は flow に残し、stock からは必要ならリンクのみにする。
- **`docs/stock/` を変更したら波及を確認する**: その stock ドキュメントに依存・関連する既存ルールや他ドキュメント (README・他の stock・`.claude/rules/`) に修正が必要ないかを確認し、必要なら同じターンで直す。

新たな置き場所を増やしたい場合は、本ルールの許可リストと `scripts/check_docs.sh` の配置検査を同時に更新する。
