# 設定値の置き場所はスコープで決める

設定項目を追加・整理するときは「その値がどのスコープのロジックに効くか」で置き場所を決める (FB 2026-07-23)。

- **メニュー (MenuKind) の属性**: 特定メニュー型のロジックにしか効かない値。閾値・本数・目標 SPM に加え、基準時間ガード・フェーズタイムアウトもここ (タイムアウトの根拠が閾値ギャップから逆算される = メニューの構造に従属するため)。phone のメニュー編集画面で変更する。
- **プロファイル/共通設定 (SettingItem)**: メニューに属さない値のみ。プロファイル (年齢・安静時心拍・身長) と、メニュー横断で一定にしたいユーザ嗜好 (提案閾値)。
- 迷ったら「メニューごとに変えたくなる合理的シナリオがあるか」で判定する。ある → メニュー属性。
- 理論定数 (例: maxBaseline = 心拍 onset kinetics 上端) は編集 UI を持たせず SessionConfig のデフォルトに留める。
