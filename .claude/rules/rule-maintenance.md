# ルール運用 (メタルール: ルールの書き方)

「補足・指摘を受けたらルールに反映する」という常時適用の方針は CLAUDE.md「常に守ること」にある。本ルールは **どう書くか** を定める (ルール / CLAUDE.md を編集するとき適用)。

- **`paths` 付きルールには「そのパスのファイルを編集するとき気を付けること」だけを書く。ルールは関連トピックの解説場所ではない。** `paths` 付きルールは該当ファイルが文脈に入った後に初めてロードされるので、編集より前に効くべき内容をそこに書いても手遅れで機能しない (例: kotlin.md に「主開発言語は Kotlin」と書いても、Kotlin ファイルを開いた時点では既に Kotlin を選んでいる)。
- 「編集前に効くべき内容」は次に置く:
  - 編集パスに依存せず常に効く原則 (協働モデル・コメント規約・補足の反映方針など) → CLAUDE.md か `paths` 無しルール。
  - 確定した要件・技術選定・運用方針 (どの言語/ツール/API を採るか等) → `docs/stock/`。必要なら該当 path ルールから「着手前に読む」と明示参照する (下記「stock ドキュメントをルールから読み込ませる」)。
- ルールは ADR を介さず本文に直接書く。
- 1 ルール = 1 関心事。新しい論点はファイルを新設する。
- frontmatter に `description` と適用 `paths` を書く。Claude Code は `paths` 無しのルールを起動時に、`paths` 付きを該当ファイルが文脈に入った時に自動ロードする (正式仕様)。中央目次は作らない。
- ルールを追加・変更したら、整合しない既存ファイル・コードを同じターンで直す。

## CLAUDE.md は最小限の入り口に保つ

CLAUDE.md には、常に適用される本当に重要なルールと、詳細ルールへの入り口だけを簡潔に書く。肥大化させない。

- 書いてよい: 協働モデル等の常時適用ルール、横断的な最重要原則。
- 書かない: 個別ルールの目次/一覧表 (肥大化する)、特定パスにだけ効くルールの本文 (それは `.claude/rules/` に置く)。

## stock ドキュメントをルールから読み込ませる

ルールファイル (`.claude/rules/*.md`) は `@import` を解釈しない (CLAUDE.md 専用)。Markdown リンクも自動ロードされない。stock ドキュメント (`docs/stock/`) をルールに連動させるには次のどちらかを使う。

- **常に文脈に入れたい stock ドキュメント** → `CLAUDE.md` に `@docs/stock/<file>.md` を import する (起動時に無条件ロード)。肥大化するので、本当に常時必要なものだけに限る。
- **特定の作業時だけ必要な stock ドキュメント** → 該当する path スコープのルールに「この作業では `docs/stock/<file>.md` を読んでから着手する」と**明示的な指示**として書く。ルールが `paths` で条件ロードされ、その指示に従って on-demand に Read される。リンクを貼るだけでは読み込まれないので、必ず「読む」と指示すること。
